create a release signed apk file using Gradle

Google发布的Android Studio 1.0版本中的一些新特性，其中就有Manifest Merger的更新，manifest-merger官方文档，从文档看出使用Placehodler Support特性就可以来完成完成我们的任务。
多渠道打包的大概实现办法是（比如UMeng统计）: 动态的更改AndroidManifest.xml文件中的一个属性值（渠道号），从而来标识不同的渠道。
Gradle的Product flavors特性可以很方便的实现功能，简单的说下实现方式：不同的flavor指定不同渠道的AndroidManifest.xml文件。这种方式有很大的弊端，重复性工作太多了，只为了更改其中的一个属性而要维护太多的AndroidManifest.xml文件。欣喜的是Android Studio 1.0 提供了更强大的Manifest Merger（）功能。
