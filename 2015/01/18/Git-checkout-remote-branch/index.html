Git - checkout remote branch

通常情况下，你在remote仓库基于master做出develop分支后，会将develop分支clone至local仓库

然后做完事情后，可能要合并至remote仓库的master分支上，首先先查看下branch的信息


确定remote存在master分支后，使用


这样就可以将remote的master分支映射至本地，然后就可以checkout至master分支，merge之前的develop分支

merge修改冲突之后，保证程序一切正常后，记得做一个Tag

然后记得push到remote仓库，如果是github，那么记得push时加上tags标记
否则tag不会被上传
